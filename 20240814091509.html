Fundamentals on UNIX Systems

  • The design of Unix-based systems, which includes Linux distributions like Ubuntu, is deeply rooted in the following core principles:

    1. The Unix Filesystem: The Unix filesystem employs a hierarchical structure of directories (folders within folders) that provides an intuitive way to organize files and directories.
      • It also simplifies access to hardware devices like disks and tapes by representing them as files within the directory structure, making them easier to manage.
    2. Input/Output Redirection and Pipes: The ability to redirect the output of one command to a file (using >) or to the input of another command (using |) is a defining characteristic of Unix-based systems.
      • This modularity allows users to chain together simple commands to perform complex tasks, promoting code reuse and flexibility.
    3. Portability: Unix was designed to be portable across different hardware platforms.
      • This was achieved by abstracting hardware-specific details into device drivers, allowing applications to interact with a consistent interface regardless of the underlying hardware.
      • The use of the C programming language further enhanced portability, as C code can be compiled to run on various architectures.

  • These principles have had a profound impact on the evolution and success of Unix-based systems:

    • Modularity and Code Reuse: The combination of the filesystem structure, I/O redirection, and pipes encourages a modular approach to software development.
      • This has led to a rich ecosystem of small, specialized utilities that can be combined in countless ways, fostering code reuse and enabling users to build powerful solutions from existing components.
    • Portability and Flexibility: The focus on portability has allowed Unix-based systems to adapt to a wide range of hardware platforms, from mainframes to embedded devices.
      • This flexibility has been instrumental in the widespread adoption of Linux, as it can run on virtually any hardware configuration.
    • Collaboration and Community: The open nature of Unix-based systems, coupled with the tools for sharing and collaborating on code, has fostered a strong sense of community and collaboration among developers.
      • This has led to rapid innovation and continuous improvement of the software, as well as the development of a vast ecosystem of open-source software.

UNIX-based Systems

  • The main UNIX-based systems available today are:

    • macOS: The operating system used by Apple computers, which is built on a derivative of BSD (Berkeley Software Distribution) UNIX.
    • Linux: The open-source operating system that has become widely popular and is used in a vast array of devices, from servers to smartphones.
      • Some Linux distributions are:
        • Debian: One of the oldest and most influential Linux distributions, known for its stability and adherence to free software principles. It serves as the foundation for many other distributions, including Ubuntu.
        • Ubuntu: Perhaps the most popular Linux distribution, sponsored by Canonical Ltd. It is known for its user-friendliness, regular release cycle, and strong community support. It is derived from Debian and offers various flavours for different use cases.
        • Mint: Another popular Debian-based distribution that emphasizes ease of use and comes with various desktop environments and pre-installed software.
        • Kali Linux: A Debian-based distribution specifically designed for penetration testing and security auditing. It includes a wide range of tools for network analysis, vulnerability assessment, and exploitation.
        • CentOS: A community-supported distribution that is closely aligned with Red Hat Enterprise Linux (RHEL). It offers a stable and reliable platform for server environments and is often used as a free alternative to RHEL.
        • Fedora: A community-driven distribution sponsored by Red Hat. It is known for its focus on innovation and cutting-edge technologies, typically serving as a testing ground for new features that may eventually be incorporated into RHEL.
        • KNOPPIX: A live CD Linux distribution that can be booted directly from a CD or USB drive without requiring installation on the hard drive. It is typically used for system rescue, data recovery, or trying out Linux without making any permanent changes to the computer.
        • Gentoo: A highly customizable Linux distribution that allows users to compile their own software packages from source code, providing a high degree of control over the system's configuration and performance.
        • Red Hat Enterprise Linux (RHEL): This is Red Hat's flagship product, a commercially supported Linux distribution designed for enterprise environments. It emphasizes stability, security, and long-term support, making it a popular choice for businesses and organizations that require a reliable and well-maintained operating system.
    • BSD: The Berkeley Software Distribution, which is another family of open-source Unix-like operating systems.
      • The three major BSD projects include: FreeBSD, NetBSD, and OpenBSD.
    • Android:
      • Linux Kernel: Android's core is built upon a modified version of the Linux kernel. This kernel handles fundamental tasks such as memory management, process scheduling, and hardware interaction.
      • Open Source: Like Linux, Android is largely open source, allowing developers and manufacturers to modify and customize it for their specific needs.
      • Shared Lineage: Both Android and Linux trace their roots back to the Unix philosophy of modularity, portability, and powerful command-line tools.

Structure

  • UNIX

    • Kernel: The kernel is the core of the operating system, responsible for managing hardware resources, handling processes, and providing essential services. It acts as the bridge between the hardware and the software, ensuring that applications can interact with the underlying system in a controlled and secure manner.
    • Shell: The shell is a command-line interface that allows users to interact with the system by typing commands. It provides a powerful and flexible way to execute programs, manage files, and configure the system.
    • Utilities and Applications: UNIX-based systems come with a rich set of utilities and applications that perform various tasks, from file management and text editing to network communication and system administration. These tools are designed to be modular and composable, allowing users to combine them in powerful ways to achieve their goals.
    • Filesystems: The filesystem is the structure in which all the information on the computer is stored. It is organized hierarchically, with the root directory (/) at the top. This structure provides a logical and organized way to store and access files and directories.
    • Processes: Processes are instances of running programs. The operating system manages these processes, allocating resources such as CPU time and memory, and ensuring that they can run concurrently without interfering with each other.
    • Users and Groups: The system supports multiple users and groups, allowing for secure and controlled access to system resources. Each user has their own home directory and can be assigned to one or more groups, which can be used to grant shared access to files and directories.
    • Services: Services, or daemons, are background processes that provide various functionalities, such as network connectivity, printing, and system logging. These services are typically started at boot time and run continuously in the background, responding to requests from users and other systems.
